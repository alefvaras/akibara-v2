## Definition of Done universal

Todo item del BACKLOG marca DONE solo cuando cumple:

1. Cambio commiteado a `main` con mensaje claro (qué + por qué + rollback path).
2. Smoke test post-cambio ejecutado y passing (mínimo: home HTTP 200 + producto test load + checkout flow basic).
3. Documentación inline si requerida (PHPDoc para funciones públicas nuevas).
4. Si toca emails: smoke send a `alejandro.fvaras@gmail.com` + Gmail MCP verify llegada + Brevo dashboard verify event registered.
5. Si toca prod (server): snapshot pre-cambio en `.private/snapshots/YYYY-MM-DD-<item-id>.tar.gz` + DOBLE OK explícito.
6. Si toca DB: backup mysqldump pre-cambio + verify post-cambio + rollback documentado <30 min.
7. Si toca branding/visual: mockup Figma aprobado linked en commit message + LambdaTest cross-browser passed.
8. Si toca pricing/cupones: verify NO se modifica `_sale_price`/`_regular_price`/`_price` directos + solo via WC_Coupon API.

## Architects review per sprint (RACI)

Cada sprint pasa por 3 gates de arquitectura:

| Gate | Cuándo | Quién (Responsible) | Output |
|---|---|---|---|
| **PRE-review** | Día 1 sprint, antes de implementar | mesa-15-architect-reviewer | `audit/sprint-{N}/ARCHITECTURE-PRE-REVIEW.md` |
| **MID-checkpoint** | Mid-sprint si dura >5 días | mesa-15 | `audit/sprint-{N}/ARCHITECTURE-MID-CHECKPOINT.md` |
| **POST-validation** | Último día sprint, antes de cerrar | mesa-15 + mesa-22 + mesa-02 | `audit/sprint-{N}/ARCHITECTURE-POST-VALIDATION.md` |
| **Quarterly audit** | Cada 3 sprints | mesa-15 | `audit/quarterly/ARCHITECTURE-AUDIT-Q{N}.md` |

Detalles completos del workflow en `audit/round2/ARCHITECTURE-ROBUSTNESS-MESA.md` sección 5.

**Quarterly re-audit triggers (escalar a mesa de 22+ × 7 rondas):**
- Tráfico estable >50 clientes activos/mes
- Revenue $1M CLP+/mes
- Aparece 4to o 5to plugin custom (señal fragmentación)
- Decisiones arquitectónicas controversiales aparecen
- Akibara contrata 2do dev
