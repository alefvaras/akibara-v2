A11y rules (CRITICAL) 
// Cell B implementa wp-content/plugins/akibara-marketing/assets/js/modal.js
//
// 1. role + aria
//    role="dialog" aria-modal="true" aria-labelledby=""
//    backdrop has aria-hidden="true"
//
// 2. Focus trap (cycle Tab within modal):
//    const focusable = modal.querySelectorAll(
//      'button, [href], input, textarea, select, [tabindex]:not([tabindex="-1"])'
//    );
//    on Tab key → if last+shift OR first+!shift, wrap to opposite end
//
// 3. Open behavior:
//    - Save document.activeElement (trigger) to restore on close
//    - inert attribute on body siblings (or aria-hidden=true on root)
//    - Move focus to first focusable in modal (or close button)
//
// 4. Close behavior:
//    - Escape key → close
//    - Click backdrop → close (configurable per variant; review variant disables this)
//    - Click .aki-modal__close → close
//    - On close: remove inert/aria-hidden, return focus to trigger
//
// 5. prefers-reduced-motion:
//    - Disable scale/fade animation, just show/hide
Trigger logic (Cell B implementa) 
// Newsletter (variant 1):
//   - Time delay: 30s after page load OR scroll depth >50%
//   - Once per 7 days (localStorage akibara_modal_newsletter_shown)
//   - NOT on /checkout, /mi-cuenta
//
// Discount promo (variant 2):
//   - Exit-intent: mouseleave with clientY <= 0
//   - Once per session
//   - NOT logged-in users (already have welcome notice item 6)
//
// Review request (variant 3):
//   - Order status delivered + 7 days passed + customer hasn't reviewed
//   - Account dashboard ONLY (not popup on shop pages)
//   - Click "Más tarde" → defer 30 days
//   - Submit → POST /wp-json/akibara/v1/reviews + close modal
