Brevo wiring (Cell B implementa) 
// REST endpoint: POST /wp-json/akibara/v1/newsletter-subscribe
//   { email: "user@example.cl", list_id: 4 }
//
// Cell B handler (wp-content/plugins/akibara-marketing/includes/api-newsletter.php):
//
// 1. Validate email server-side (filter_var FILTER_VALIDATE_EMAIL)
// 2. Call Brevo API:
//    POST https://api.brevo.com/v3/contacts
//    headers: api-key (from wp-config-private.php BREVO_API_KEY)
//    body: {
//      email,
//      listIds: [4],                    // newsletter list
//      attributes: { OPT_IN_SOURCE: 'footer-newsletter', OPT_IN_DATE: now() },
//      updateEnabled: false              // 409 if duplicate
//    }
// 3. If 201 → respond { status: 'success' }
//    If 400 invalid email → { status: 'error', code: 'invalid_email' }
//    If 409 duplicate → { status: 'error', code: 'duplicate' }
//    Brevo automatically envía DOI confirmation email (configured per-list)
//
// 4. Frontend JS (vanilla):
//    - on form submit: prevent default, fetch POST
//    - if success: replace form with .aki-newsletter__feedback--success + GA4 event
//    - if error: add .aki-newsletter-form__input--error + render feedback
//
// 5. Welcome flow: triggered AFTER user click confirm in DOI email
//    Brevo webhook → Cell B route → set customer meta welcome_notice_pending=true
