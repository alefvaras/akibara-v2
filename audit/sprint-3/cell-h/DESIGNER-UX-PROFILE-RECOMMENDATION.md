# Perfil Diseñador / UX freelance recomendado — Akibara Phase 2

**Fecha:** 2026-04-27
**Contexto:** Sprint 3 paralelo cerró con specs + design tokens (Cell H) + HTML/CSS prototypes (fallback Figma MCP bloqueado). Phase 2 contempla designer freelance que tome el work-in-progress y lo lleve a producción visual completa.

---

## TL;DR

**Buscar:** UX/UI designer mid-senior, español nativo (Chile o LATAM), 3+ años e-commerce, Figma fluent (Variables + Components), WCAG AA awareness, portfolio con 2+ shops WooCommerce o Shopify.

**NO buscar:** generalist graphic designer, "branding agency", junior sin portfolio ecommerce, alguien que no diferencia UX de UI, alguien que cobra por "intuición" sin métricas.

**Onboarding:** 1 brief + entrega de assets ya producidos (tokens.css, UI specs, HTML prototypes, Figma file con 45 tokens importados) → designer empieza con foundation lista.

## 3. Plataformas para hiring

**Recomendadas (orden por ROI Chile/LATAM):**

1. **Workana** ([workana.com](https://workana.com)) — LATAM-focused, español nativo, bidding system. Filtro: Chile/Argentina, rating 4.8+, 50+ proyectos. Esperar ~$1500-3500 USD por sprint Phase 2.

2. **LinkedIn Jobs Chile** — designers in-house que freelancean. Mensajes directos via Sales Navigator si se justifica.

3. **Behance** ([behance.net](https://behance.net)) — buscar portfolios "ecommerce manga" / "WooCommerce custom theme" / Chile location. Contactar por mensaje, propuesta de proyecto fixed-scope.

4. **Domestika** ([domestika.org](https://domestika.org)) — comunidad latina, instructores y freelancers. Más visual portfolio que LinkedIn.

5. **Upwork** ([upwork.com](https://upwork.com)) — escalable, pero Filtros Chile/Mexico/Argentina + Job Success Score 95%+. Más fácil de scale después de primer contratación.

**Evitar para Akibara scope:**
- **Toptal** — high-end ($100+/hr), overkill para 3-clientes-shop
- **99designs / DesignCrowd** — contests, no fit para ongoing partnership
- **Fiverr** — calidad inconsistente, hard to vet

## 6. Criterios de evaluación (entrevista 30 min)

**Portfolio review (15 min):**
- ✅ 2+ proyectos ecommerce reales (no concepts)
- ✅ Mobile screenshots presentes (no solo desktop hero shots)
- ✅ Diferencia entre proyecto early-stage vs polished (mostrar progresión)
- ✅ Variants/states visibles (no solo final hero)

**Conversación (10 min):**
- Pregunta: "¿Cómo decidiste el contrast ratio del CTA principal en este proyecto?" — busca awareness a11y, no diseño "intuitivo"
- Pregunta: "¿Cómo manejas design tokens que se desfasen del código?" — busca workflow design ↔ dev, Code Connect awareness
- Pregunta: "Mostrame un componente con 4+ states (default/hover/focus/disabled/error)" — confirma Figma Variants fluency

**Test corto (5 min, opcional):**
- Pasa el spec del Item 1 (encargos form) en `UI-SPECS-preventas.md` y pide bocetar layout en Figma en 5 min — sin polish, solo wireframe + tokens. Evalúa fluency Figma + comprensión specs.

**Red flags:**
- ❌ Insiste en hacer "branding desde cero" cuando hay Manga Crimson v3 ya
- ❌ Cobra por "rounds de revisiones unlimited" — implica scope creep no controlado
- ❌ No pregunta sobre stack técnico (designer disconectado de dev)
- ❌ Portfolio puro Behance award-winners sin métricas conversion
- ❌ "No uso variables/tokens — prefiero copy-paste hex"

## 8. Alternativas a freelance (si presupuesto/timing no calzan)

1. **Estudiante diseño UX/UI Chile** (DuocUC, Universidad de Chile, Universidad Diego Portales — Magíster Diseño)
   - Pro: precio bajo ($500-1500), motivación
   - Con: portfolio limitado, requiere más mentoring
   - Plataforma: LinkedIn estudiantes + perfil "buscando experiencia"

2. **Designer in-house de empresa amiga** que freelance off-hours
   - Pro: validated portfolio, sólida ejecución
   - Con: capacity limitada, plazos elásticos
   - Plataforma: networking directo

3. **AI tools + agent pipeline** (lo que ya estás haciendo)
   - Pro: zero ongoing cost, fast iteration, control completo
   - Con: limitado en taste/intuition visual, requiere refinement humano para "pulse" final
   - Sprint 3 demostró que cubre 70-80% del work; el 20-30% final beneficia de designer humano

## 9. Decisión a tomar (cuándo contratar)

**Contratar Phase 2 designer cuando:**
- ✅ HTML prototypes Sprint 3.5 están done y aprobados (foundation visual lista)
- ✅ Cells A+B implementaron lógica + UI con stubs (no bloqueante para designer)
- ✅ Tienes 2-3 semanas wall-clock disponibles para review iterations
- ✅ Hay budget disponible $2K-3.5K USD

**Diferir contratación si:**
- ⏸️ Cells A+B aún tienen módulos críticos pendientes (caso actual: 9 módulos Cell B aún en scaffold)
- ⏸️ HTML prototypes son suficientes para próximo sprint deploy
- ⏸️ Tienda 3 clientes — esperar a 50+ clientes/mes para justificar inversión visual
- ⏸️ Métricas conversion actuales son aceptables (no hay revenue leak visible que designer resolvería)

**Mi lectura honesta del momentum Akibara:**
- Foundation Sprint 3 está sólida (specs + tokens + prototypes)
- El bottleneck NO es visual — es completar Cell B 9 módulos restantes (banner, brevo, popup, etc.) y resolver decisión cart-abandoned vía Brevo upstream
- Recomiendo **diferir designer Phase 2 hasta después de Sprint 4-5** (cuando Cells A+B y theme estén production-ready y tengas baseline de conversion para que designer optimice contra data, no contra "intuición")

## 10. Lectura recomendada antes de contratar

- "Lean UX" by Jeff Gothelf — mindset que se busca en el designer
- "Refactoring UI" by Adam Wathan + Steve Schoger — visual heuristics que cualquier designer mid+ debería conocer
- "Don't Make Me Think" by Steve Krug — clásico UX, baseline conversation
